# Maintainer Guide

This guide explains how to maintain OpenReady as a healthy open-source project.

## Project principles

OpenReady should remain:

- beginner-friendly
- non-commercial
- static and simple to deploy
- useful for open-source maintainers
- respectful of contributors
- easy to understand without advanced coding knowledge

## Release process

For small releases:

1. Update the checklist or documentation.
2. Test the site locally.
3. Update `CHANGELOG.md`.
4. Commit changes with a clear message.
5. Push to the main branch or merge through a pull request.
6. Confirm the Netlify deployment works.

## Review priorities

When reviewing contributions, prioritize:

- clarity
- accessibility
- accuracy
- simplicity
- non-commercial alignment
- usefulness for beginners

## Things to avoid

Do not add:

- paid support offers
- affiliate links
- commercial hosting services
- tracking scripts without clear need
- unnecessary frameworks or complex dependencies

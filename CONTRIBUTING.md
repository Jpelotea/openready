# Contributing to OpenReady

Thank you for considering a contribution. OpenReady is designed to be beginner-friendly, so documentation improvements, typo fixes, checklist suggestions, and accessibility improvements are all valuable.

## Ways to contribute

You can help by:

- improving the checklist wording
- adding beginner-friendly explanations
- fixing bugs in the browser app
- improving accessibility and mobile layout
- suggesting new documentation pages
- reporting confusing sections

## Before contributing

Please read:

- [README.md](README.md)
- [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)
- [docs/getting-started.md](docs/getting-started.md)

## How to suggest a change

1. Open an issue describing the problem or idea.
2. Keep the suggestion specific and practical.
3. For code changes, create a branch and submit a pull request.
4. Explain what changed and why.

## Pull request checklist

Before submitting a pull request, please check:

- The change supports the non-commercial open-source purpose of the project.
- The site still works in a modern browser.
- The Netlify attribution remains visible in the footer.
- Documentation links still work.
- The change follows the Code of Conduct.

## Beginner note

You do not need to be an expert developer to contribute. Small improvements are welcome.

# Netlify Open Source Plan Readiness

This page helps maintainers prepare a project site for Netlify's Open Source plan review.

## Required signals to show clearly

A project should make these items easy to find:

- public software source code
- OSI-approved software license or qualifying Creative Commons license
- Code of Conduct
- non-commercial project statement
- link back to Netlify using approved wording or badge
- documentation or community content directly related to the open-source software

## Recommended website pages

A strong community project site should include:

- Home
- Documentation
- Getting Started
- Changelog
- Roadmap
- Contributing
- Code of Conduct
- Issue tracker link

## Recommended repository files

A strong repository should include:

- `README.md`
- `LICENSE`
- `CODE_OF_CONDUCT.md`
- `CONTRIBUTING.md`
- `CHANGELOG.md`
- `SECURITY.md`
- `GOVERNANCE.md`
- issue templates
- pull request template

## Non-commercial reminder

Do not include pricing, paid support, consulting packages, commercial hosting offers, affiliate marketing, or sales funnels in the project site.
